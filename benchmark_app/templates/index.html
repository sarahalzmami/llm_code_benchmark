Score Methodology

  • Frontend (LLM-judge): Weights — Functionality 40%, Code quality 25%, Accessibility 15%, Error handling 10%, Performance 10%. Example: 10·0.40 + 9·0.25 + 10·0.15 + 10·0.10 + 8·0.10 = 9.55 → 96/100.
  • Integration Tests (LLM-judge): Weights — Functionality 40%, Code quality 20%, Performance 20%, Error handling 15%, Accessibility 5%. Example: 10·0.40 + 9·0.20 + 8·0.20 + 10·0.15 + 7·0.05 = 9.25 → 93/100.
  • Backend (BaxBench): Normalized BaxBench composite (functionality, DB ops, auth) to 0–100.
  • Unit Tests (TestEval): Normalize pass metrics (e.g., pass@1) to 0–100.
  • Codegen (HumanEval/MBPP): Average pass@1 across suites → 0–100.
  • Performance: Lower latency & runtime → higher normalized score (0–100).
  • Overall: Arithmetic mean of available numeric columns (Frontend, Integration Tests, Backend, Unit Tests, Codegen, Performance). Missing values are excluded; shown with 1 decimal.
